wisp · a clean-room rebuild

A little light that lives on your Mac.

Wisp (named for the will-o'-the-wisp, the floating light that guides travelers) rebuilds the experience Clicky demonstrated: it sits next to your cursor, sees what you see, walks you through what you're doing out loud — or works in the background and reports back on a card. This workspace documents studying the original and rebuilding it cleanly from scratch.

The load-bearing finding. The shipped product's UX is not what the original repo documents (a big floating panel with an arc-flying blue cursor). The relaunch language is minimal and cursor-adjacent: a menu-bar companion, a tiny cursor-trailing glyph, top-right task cards, and screenshot-anchored drawing. We rebuild to the relaunch language.

Two modes, one companion

Ask / guide

Talk to it. It answers with voice and draws or points directly on your screen to teach you — a live tutor for anything in any app.

Agent

"Clicky agent, …" spawns a background agent that does real work: organizing files, controlling native apps, research → CSV, even building & launching a native Mac app.

Scope of the rebuild

Tier 1 · core

Menu-bar app, push-to-talk + keyword, local Parakeet STT, screen capture, Claude Fable 5 via proxy, streamed Inworld voice, cursor glyph, task cards, completion toast, agent status stack.

Tier 2 · magic

On-screen teaching overlay (red draw + accumulating chips), screen-aware dictation into the focused app, native actions (Reminders/Calendar via EventKit).

Tier 3 · stretch

Spatial context (draw-to-direct), multi-agent concurrency, "build a Mac app" agent, Skills Library, per-user memory.

Two drawing languages

AI teaching ink

Single red #FF3B30. Crisp outlines, arrows, and pixel-precise dots + white-on-red chip labels that accumulate into a persistent legend. Mechanism: screenshot → model returns coordinates → overlay paints at absolute coords.

User draw-to-direct

Hold PTT + drag on live content. Thick translucent marker, mode-coded: coral #E8776B = "tell me about this", mint #3ECC8E = "build this". A waveform badge rides the stroke tip; the ink fades after capture.

More verified surfaces (full-campaign sweep)

Proactive nudge

A dismissible toast docked to the frontmost window, triggered by passive app recognition: "Connect Notion to Clicky?" + three concrete suggestion chips + ✕ No · 🕐 Not now · 🔗 Yes.

Drag-files-in attach

Dragging files to the top of the screen expands the panel into a drop target, resolving to per-file-type chips above an "ask HeyClicky…" input.

Agents panel

RUNNING / TODAY sections, per-task progress bars, "Open Agent" buttons — the fuller sibling of the task-card stack.

Clicky Notes (memory)

A categorized agent-memory wiki — the "memories of me" store that powers writing "in your voice". Cursor color is user-configurable (red/blue/yellow/green presets).

Type

Deliberately Mac-native — SF Pro Display / system-ui — because this is a Mac-native product. Friendly lowercase wordmark; SF Mono for timestamps and data. The type is the OS the app lives in.

Reverse-engineering the shipped app

Observable-behavior inventory only — no proprietary source is copied. The shipped app is far beyond the original repo: a voice-driven agentic assistant.

Headline finding

Same lineage, two generations. A leaked dev path in the binary (…/clicky-closed/leanring-buddy/DictationDeepgramTranscriber.swift) proves the shipped app is built from the same Xcode project as the local repo — the repo is an earlier snapshot; the shipped build is a superset with Realtime voice, a notch HUD, an agent runtime, a paywall, and feature flags.

What ships in the bundle (deep pass, verified)

  • Menu-bar only (LSUIElement), unsandboxed, com.humansongs.clicky v1.0.38; Sparkle 2.9.0 hourly updates; Sentry; PostHog analytics + feature flags gating whole surfaces (CatMode, ProactiveAgentSuggestions, TransparentPanes, UpcomingMeetings…).
  • Notch-anchored HUD is the primary surface — collapsed/expanded states, a file-drag composer, auto-dismissing text responses with image galleries. Floating cards are satellites.
  • Codex agent runtime pinned at CLI 0.132.0 (model gpt-5.6-sol, ~194MB per-arch binaries + vendored ripgrep) and a standalone Computer-Use Mach-O helper exposed as a local computer-use MCP server: snapshot-act-verify on every action, a strict never-steal-focus contract, AX-element addressing (coordinate clicks blocked).
  • Richer pointing vocabulary than the old repo: [TARGET:x,y,r:label], [HOVER], [HIGHLIGHT], [SHAPE:arrow], [SHAPE:curve] — TARGET/HOVER only for a single observable next action. Agent cursor: bezier-glide triangle, ring-ripple on landing, idle-hide ~1.5s.
  • Voice stack (shipped): always-on OpenAI Realtime mode with barge-in + per-topology AEC caching; dictation via Deepgram (the ~450ms path) plus AssemblyAI PTT; two voice-picker sets (OpenAI Realtime voices + a friendlier branded set). Our Parakeet-local STT is a deliberate divergence keeping the same UX contract.
  • Proactive agents are server-driven: a /proactive-agents endpoint feeds suggestion cards with daily slots, corner nudges, and approve/deny resolution reporting with quota gates.
  • PowerUp skills marketplace with creator attribution, a "hatching" celebration (dedicated sounds), and pinned-to-notch states. Cross-session user memory is injected as a <user_memory> system-prompt block.
  • Only the 14 bundled workflow skills are live (artifacts, build-preview, creative-studio, dev-setup-doctor, email-assistant, google-workspace, repo-operator, research-report, cua-driver, doc, pdf, spreadsheet, frontend-design, obsidian, vercel-deploy). The big integration .md catalog (Notion, Linear, Airtable, iMessage, Spotify…) is vendored from hermes-agent and inert — onboarding-picker copy only.
  • Hard paywall before agent launches (plans, billing toggle, "maxed out") — Claude Haiku generates the launch-card labels. Remappable hotkeys; onboarding skip-shake gesture; 44MB onboarding + 26MB paywall intro videos with a 5.7-minute music bed.

Voice pipeline

STT — local

Parakeet TDT 0.6B v3 (parakeet-mlx, Apple Silicon) via a Python sidecar streaming partials + finals; Silero VAD + Smart Turn V3. Fallback: Apple Speech / whisper. ~450ms feel.

TTS — cloud, proxied

Inworld inworld-tts-2, Kurzgesagt voice, streamed; fallback ElevenLabs → OpenAI TTS → AVSpeechSynthesizer. All via the Worker.

Preserved as concepts (not code) from the original repo: single shared URLSession for streaming, short-lived proxy tokens, multi-display coordinate conversion, the coordinate-tag pointing protocol.

Content strategy

The launch is a masterclass. These patterns feed a reusable content-production skill suite for ~/.claude and ~/.codex.

The dual-channel system

Founder flagship (@FarzaTV)

Authority + reach. 4K landscape. "Today we're shipping X." Ships exactly one crisp new primitive per drop, with a live, un-cut demo.

Mascot diary (@heyclicky)

Personality + relatability. Vertical clips in the product's own first-person voice. Keeps cadence between big drops and makes the brand lovable.

The flagship hook formula

  • Bold claim — "we built an AI that can draw on your screen."
  • Acknowledge the crowd — "many products already do this."
  • "So we went further" — the pivot to the unique mechanism.
  • The magic, shown live — real demo, escalating stakes (fun → utility → high-stakes).
  • "Demo:" + candid close — often a "still a bit buggy" admission that builds trust.

Mascot channel — three repeating templates

  • Sprint update: "everything we're shipping" with a 3–5 person cast passing one mic hand-to-hand, big synced captions, version number as an on-camera prop ("v1.0.11" on a whiteboard).
  • Founder-solo narrative: fixed outdoor seat, 1–2 screen-recording proof cutaways, ends on a "heyclicky.com" card.
  • Bare feature demo: raw screen recording, no cast, 4–15 seconds ("drag your files in").

Craft notes

  • Format split: 4K landscape for founder demos, 1080×1920 vertical for mascot clips; widescreen framed within a vertical-safe column for repurposing.
  • Consistent "keynote" compositing — inset app windows over a warm desert backdrop — unifies wildly different apps into one visual system.
  • Karaoke captions in 3–6 word chunks; brisk 1–3s cuts alternating talking-head and screen B-roll.
  • Reveals build across the campaign: pointer → drawing → spatial context → dictation. Each drop earns the next.

Testing evidence

  • /tts verified end-to-end: local wrangler dev → Inworld inworld-tts-2 (Kurzgesagt voice) returned a real 128kbps/48kHz MP3, HTTP 200.
  • Graceful fallback verified: a bad Inworld voiceId produced a 404 from Inworld and the Worker transparently served ElevenLabs audio, still HTTP 200.

Blockers needing the user (all remaining work is gated here)

  • Runtime app verification — the app compiles clean but its behavior needs macOS permission grants: run swift run (or open the package in Xcode), then grant Microphone, Screen Recording, and Accessibility in System Settings → Privacy & Security. Terminal xcodebuild was deliberately never run (it invalidates TCC grants).
  • Cloudflare deploy — wrangler is unauthenticated; run npx wrangler login in worker/, then set the three secrets and npx wrangler deploy. (/tts already verified end-to-end via local wrangler dev.)
  • /chat in production — provide an Anthropic API key (or mint a Vercel AI Gateway key in the dashboard) as the ANTHROPIC_API_KEY worker secret.
  • Parakeet model download — first run of voice-sidecar/parakeet_stt.py pulls the ~600MB Parakeet TDT 0.6B v3 model; needs pip install -r requirements.txt.
